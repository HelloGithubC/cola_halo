# COLA Halo OMP 内存分析

## 分析参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `nc` | 1024 | 粗网格数 |
| `nc_pm_factor` | 3 | PM网格因子 |
| `Ngrid` (PM网格) | 3072 | nc × nc_pm_factor |
| 粒子总数 | 1024³ = 1,073,741,824 | 约10.7亿粒子 |
| 单精度float | 4 bytes | |
| fftwf_complex | 8 bytes | 2×float |

以下分析假设 `np_alloc_factor = 1.0`。

## 1. 粒子数据 (`allocate_particles()` in mem.c)

```c
np_alloc = np_alloc_factor * nc * nc * (local_nx + 1)
         = 1.0 × 1024 × 1024 × 1025 ≈ 1,074,794,496
```

| 数据结构 | 大小计算 | 内存占用 (单节点) |
|----------|----------|-------------------|
| `Particle` 结构 | 60 bytes/粒子 (x[3]+dx1[3]+dx2[3]+v[3]+id) | **60 GB** |
| `force` (float3) | 12 bytes/粒子 | **12 GB** |
| **粒子总计** | 72 bytes/粒子 | **72 GB** |

## 2. 共享内存 mem1 (`allocate_shared_memory()` in mem.c)

mem1 用于 LPT网格 / PM网格 / FoF，取三者最大值：

### 2.1 LPT网格需求
```c
ncomplex_lpt = 12 × size_lpt_one
size_lpt_one ≈ nc³ / 2 (复数压缩存储)
size_lpt = 12 × (1024³/2) × 8 = **51.5 GB**
```

### 2.2 PM网格需求
```c
size_pm_one ≈ Ngrid³ / 2
size_pm = (3072³/2) × 8 = **116.1 GB**
```

### 2.3 FoF内存需求
```c
nNodes ≈ 2 × np_alloc (KD树节点)
size_fof ≈ sizeof(KDN) × nNodes + sizeof(int) × np_alloc × 3 + ...
        ≈ 40 × 2.15e9 + 12 × 1.07e9 + ...
        ≈ **~100 GB** (粗略估计)
```

**mem1 大小** = max(LPT, PM, FoF) ≈ **116 GB**

## 3. 共享内存 mem2

```c
// density_k: (Ngrid/2+1) × Ngrid × local_ny
size_density_k = (3072/2+1) × 3072 × 3072 × 8 = **116.1 GB**

// snapshot: ParticleMinimum × np_alloc (28 bytes/粒子)
size_snapshot = 28 × 1,074,794,496 = **30.1 GB**

// mem2 = max(density_k, snapshot)
mem2 ≈ **116.1 GB**
```

## 4. PM缓冲区 (`pm_init()` in pm.c)

```c
nbuf ≈ np_alloc_factor × ncp² + 5 × sqrt(...)
     ≈ 1024² + 5×1024 ≈ 1,055,000
size_buf = (sizeof(float)×6 + sizeof(int)) × nbuf ≈ **30 MB** (可忽略)
```

## 5. 内存汇总 (单节点)

| 内存区域 | 大小 | 说明 |
|----------|------|------|
| 粒子数据 | **72 GB** | 不可复用，持续占用 |
| mem1 | **116 GB** | LPT/PM/FoF复用 |
| mem2 | **116 GB** | density_k/snapshot复用 |
| PM缓冲区 | ~30 MB | 可忽略 |
| **总计** | **~304 GB** | 单节点峰值 |

## 6. 多节点分布式情况

假设使用 **N** 个MPI进程均匀分布：

| 内存区域 | 单进程占用 | 分布方式 |
|----------|------------|----------|
| 粒子数据 | 72 GB / N | 按x方向分片 |
| mem1 | 116 GB / N | PM网格按x方向分片 |
| mem2 | 116 GB / N | density_k按y方向分片 |
| **总计** | **304 GB / N** | |

**节点数示例：**

| 节点数 | 单节点内存需求 | 适用场景 |
|--------|----------------|----------|
| 1 | ~304 GB | 超大内存节点测试 |
| 16 | ~19 GB | 小规模测试 |
| 64 | ~4.75 GB | 中等规模生产 |
| 128 | ~2.4 GB | 推荐生产配置 |
| 256 | ~1.2 GB | 大规模生产 |

## 7. 内存优化建议

1. **降低 `nc_pm_factor`**: 从3改为2，PM网格从3072降至2048，内存减少约70%

2. **增加MPI进程数**: 内存需求与进程数成反比

3. **调整 `np_alloc_factor`**: 如果粒子分布均匀，可适当降低（默认1.0已有余量）
