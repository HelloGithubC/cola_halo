## 2026-5-10 第一阶段分析：粒子化操作差异

### 分析目标
调查C版本主程序中是否存在类似FML版本的独立"粒子化"步骤，以及测试程序是否正确应用了该步骤。

### 关键发现
**C版本主程序中不存在独立的"粒子化"步骤**。LPT位移场赋值与粒子位置初始化在同一个函数 `lpt_set_displacement` 中一步完成（`src/lpt.c` lines 539-568）。

### 核心概念澄清：粒子化 ≠ 位移场插值

**容易产生的误解**：既然两个版本都是"规则网格粒子化"（粒子初始位置规则排列），为什么FML版本还会有CIC平滑效果？

**关键区分**：
1. **粒子化**：决定粒子初始拉格朗日位置 \(\mathbf{q}\) 如何排列（网格中心、网格点、或其他方式）
2. **位移场赋值**：决定如何从离散位移场 \(\Psi(\mathbf{q})\) 获得粒子位移值

**平滑效果的来源**：不是粒子化方式本身，而是**位移场赋值时是否需要插值**。如果粒子位置与位移场采样点重合，可以直接读取；如果不重合，则需要插值，而插值核在Fourier空间对应窗口函数，会抑制高 \(k\) 模式。

### 两个版本的核心差异

| 方面 | C版本 | FML版本 |
|------|-------|---------|
| **粒子初始位置** | 网格中心 `(i + 0.5) * dx` | 网格点 `i * dx` |
| **位移场采样点** | 网格中心（与粒子位置重合） | 网格中心（与粒子位置不重合） |
| **位移赋值方式** | 直接读取（无插值） | CIC插值 |
| **是否引入额外窗口函数** | 否 | 是（CIC窗口函数） |

**关键点**：C版本的粒子位置与位移场采样点完全重合，因此可以直接按网格索引读取位移值，不引入额外插值窗口；FML版本的粒子位置在网格点，而位移场采样在网格中心，两者错开半格，必须插值。

### 代码证据

**C版本（`src/lpt.c` lines 540-552）：**
```c
// 粒子初始化在网格中心
x[0] = (Local_x_start + i + 0.5f) * dx;
x[1] = (j + 0.5f) * dx;
x[2] = (k + 0.5f) * dx;

// 位移场直接从网格读取，无插值
// 注意：粒子索引 (i,j,k) 与位移场索引完全对应
float dis = disp[axes][(i * Nmesh + j) * (2 * (Nmesh / 2 + 1)) + k];

// 最终位置一步计算
p->x[axes] = x[axes] + dis*Dplus - 3.0/7.0*D20*dis2*D2;
```

**FML版本（`FML/FML/NBody/NBody.h` lines 836-840）：**
```cpp
// 使用CIC插值将位移场赋值给粒子
// 粒子位置（网格点）与位移场采样点（网格中心）不重合，必须插值
FML::INTERPOLATION::interpolate_grid_vector_to_particle_positions<N, T>(
    Psi_nLPT_vector, part.get_particles_ptr(), part.get_npart(),
    displacements_nLPT, interpolation_method);  // interpolation_method = "CIC"
```

### 为什么CIC插值会抑制高k功率？

CIC插值核在Fourier空间对应窗口函数 \(W(\mathbf{k})\)，对于三维情况：

\[
W(\mathbf{k}) = \prod_{i=1}^{3} \left[\frac{\sin(k_i dx/2)}{k_i dx/2}\right]^2
\]

这个窗口函数的特性：
- 小 \(k\)：\(W(\mathbf{k}) \approx 1\)（低频模式不受影响）
- 大 \(k\)：\(W(\mathbf{k}) < 1\)（高频模式被抑制）
- 接近Nyquist频率：\(W(\mathbf{k}) \to 0\)（最高频模式被完全压制）

因此，**CIC插值本质上是一个低通滤波器**，会系统性压低高 \(k\) 位移功率。

### 对功率谱差异的影响

这个差异是导致两个版本功率谱系统性差异的主要原因：

1. **FML版本在 k > 1.5 h/Mpc 处截断**：CIC插值引入了额外的窗口函数，抑制了小尺度（高k）功率，导致位移场功率谱在高k处下降过快，甚至出现负值。

2. **C版本延伸到Nyquist极限**：粒子位置与位移场采样点重合，直接读取位移值，不引入额外插值窗口，保持了小尺度信息的完整性，因此功率谱能够延伸到更高的k值。

### 测试程序对应性验证

C版本测试程序（`test/test_lpt_ps.c`）完全沿用主程序的 `lpt_set_displacement()` 函数，不存在遗漏任何步骤的问题。两个测试程序的实现分别对应各自主程序的行为。

### 下一步验证建议

为了确认这个差异是功率谱差异的根本原因，建议进行以下对照实验：

1. **修改FML版本**：将粒子初始化改为网格中心位置，并禁用CIC插值，直接读取位移场
2. **或者修改C版本**：将粒子初始化改为网格点位置，并启用CIC插值

通过让两个版本的实现方式一致，可以验证粒子位置初始化差异是否是功率谱差异的主要来源。

## 2026-5-10 第一阶段分析：LPT结束时粒子位置初始化差异对后续COLA演化的影响

### 分析目标
研究两版在LPT后对粒子初始化的策略差异（C版：网格中心直接读取位移；FML版：网格点+CIC插值）会如何影响后续COLA非线性演化，并检查两版是否各自存在缓解机制。

### 核心结论

**初步判断**：两种LPT后粒子初始化策略的差异主要影响初始小尺度采样方式和高$k$位移平滑程度，但对后续COLA非线性演化的影响更可能是次级的，而非灾难性的。原因是两边的COLA演化都显式保留/减去LPT轨道项，只对“残差”做PM演化，因此大部分大尺度和准线性动力学由LPT主导，PM只修正偏离LPT的部分。

### 两版COLA非线性演化的缓解机制对比

| 机制 | C版本 | FML版本 |
|------|-------|---------|
| **LPT位移存储** | 粒子结构存`dx1`,`dx2` | 粒子结构存`Psi_1LPT`,`Psi_2LPT` |
| **COLA frame初速度** | 初始化为0 | 初始化为0 |
| **Kick时LPT减法** | 显式减去LPT加速度贡献 | 在`cola_kick_drift`中显式处理 |
| **Drift时LPT加回** | 显式加回LPT轨道推进 | 在`cola_kick_drift`中显式处理 |
| **PM力计算** | CIC质量赋值，自带小尺度平滑 | 可配置density_assignment_method，与初始插值自洽 |

**结论**：两边的非线性演化都存在与各自初始化策略相匹配的COLA机制，用于缓解LPT后粒子分布差异对最终结果的影响。

### 对最终结果差异的预期

- **大尺度与中尺度**：差异大概率不显著，因为LPT主导的大尺度增长两边一致，且COLA专门强化大中尺度轨道精度。
- **最可能保留差异的尺度**：接近Nyquist的高$k$尾部、非常早形成的小质量halo、对初始细节敏感的短距离统计量。

### 关键参数辨析：如何设置FML参数以尽量接近C版

若目标是让FML/C++版本的COLA演化策略尽量接近当前C版本，以下参数应设置为：

```ini
simulation_use_cola = true
simulation_use_scaledependent_cola = false
simulation_enforce_LPT_trajectories = false
simulation_cola_LPT_order = 2
```

#### 参数含义与设置理由

| 参数 | 含义 | 推荐设置 | 理由 |
|------|------|----------|------|
| `simulation_use_cola` | 是否使用COLA算法 | `true` | C版就是COLA，不开不是同类算法 |
| `simulation_use_scaledependent_cola` | LPT主轨道是否允许$k$依赖 | `false` | C版LPT推进是统一时间增长因子，无$k$依赖 |
| `simulation_enforce_LPT_trajectories` | 是否强制只走LPT轨道（关闭PM非线性力） | `false` | C版会计算PM力并做残差kick/drift，不是纯LPT播放器 |
| `simulation_cola_LPT_order` | LPT阶数 | `2` | C版当前是2LPT COLA（存`dx1`和`dx2`） |

#### `simulation_enforce_LPT_trajectories` 的深层含义

该参数控制是否关闭PM非线性修正：
- `true`：不计算PM力，粒子只沿LPT轨道前进
- `false`：允许PM非线性力修正LPT轨道

在当前比较任务中，若设为`true`，相当于测试“只有LPT、不要非线性修正”时差异会怎样，此时初始化策略差异最不容易被后续动力学缓解。这可以作为诊断开关使用，但正常比较时应设为`false`。

#### `simulation_use_scaledependent_cola` 的深层含义

该参数控制LPT主轨道的推进是否依赖尺度$k$：
- `true`：不同尺度模式有不同的LPT推进幅度，适用于modified gravity等场景
- `false`：所有尺度共用同一时间增长因子

C版的LPT推进是简单的非$k$依赖形式，因此对齐时应设为`false`。
